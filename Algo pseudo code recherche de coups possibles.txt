(Pierre) Pseudo code recherche coups possibles

On ajoute des choses :
-        Une structure PositionPion, avec un X et un Y
o   Un array de PositionPion de taille 14, pour tous les Pions


Ensuite, on va faire une boucle de taille 14, pour tester chaque pion.
LE PRINCIPE EST LE SUIVANT :

-        Chaque groupe de 1, 2 ou 3 pions a 4 coups possibles :
o   Vers le haut
o    Vers le bas
o   Vers la gauche
o   Vers la droite

-        Cas particulier : gestion des bords.
o   Si le pion étudié est dans un bord, on dit que le mouvement vers le bord associé est impossible.

Dans le cas d’un groupe de 2 ou de 3, on aura bien deux mouvements en ligne, et deux latéraux.

-        Pour chaque coup, 3 cas de figures :
o   Si le coup est possible : bah c’est bon
o   Si le coup n’est PAS possible, c’est qu’une bille bloque le mouvement. Deux cas :
§  Si la bille n’est pas une bille du joueur.
·       On sait que le coup n’est pas valide.
·       Alors on abandonne le coup, et on passe au suivant.

§  Si la bille est de la même couleur :
·       Dans ce cas, on passe dans la configuration (n+1) billes, et on refait les 4 coups.
·       C’est comme ca qu’on fait le tour des groupes de 2 ou 3 billes qui existent.



-        Au final :
-        On étudie chaque bille, selon la fonction ci-dessus.
